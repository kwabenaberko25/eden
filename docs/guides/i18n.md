# Internationalization (i18n) 🌍

Eden provides native support for multi-lingual applications using the standard `gettext` industry format.

## Configuration

Set your supported locales and default language in your `.env` or app configuration.

```text
SUPPORTED_LOCALES=en,fr,es,de
DEFAULT_LOCALE=en
```

## The Middleware

Enable the locale middleware to automatically detect the user's preferred language via the `Accept-Language` header or a session key.

```python
app.add_middleware("i18n")
```

---

## Translating Content

### In Python Code

Use the `_()` function (alias for `gettext`) to mark strings for translation.

```python
from eden.i18n import _

async def welcome(request):
    message = _("Welcome to our garden!")
    return {"message": message}
```

### In Templates

In templates, use the standard double-curly braces for translations.

```html
<h1>{{ _("Welcome to our garden!") }}</h1>

{{ _("You have {count} message.").format(count=n) }}
```

### Pluralization

Eden supports pluralization via `ngettext`.

```html
{{ ngettext("You have {count} message.", "You have {count} messages.", n).format(count=n) }}
```

### Formatting Filters

| Filter | Description | Example |
| :--- | :--- | :--- |
| `date` | Locale-aware date. | `{{ task.due_at\|date }}` |
| `time` | Locale-aware time. | `{{ task.due_at\|time }}` |
| `number` | Thousand separators. | `{{ 1500000\|number }}` |
| `currency` | Localized symbol. | `{{ 50\|currency }}` |

## Managing Catalogs

Eden looks for translation files in your project's `locale/` directory.

```text
my_project/
└── locale/
    ├── fr/
    │   └── LC_MESSAGES/
    │       └── messages.po
    └── es/
        └── LC_MESSAGES/
            └── messages.po
```

### Compiling Translations

Use the Eden CLI to extract and compile your translation strings.

```bash
# Extract strings to .pot file
eden i18n extract

# Compile .po files to binary .mo files
eden i18n compile
```

## 💤 Lazy Translations

When defining models or form schemas, strings are evaluated at import time. To allow the language to change dynamically per request, use `lazy_gettext` (aliased as `_l`).

```python
from eden.i18n import _l

class Category(Model):
    name: str = field(label=_l("Category Name"))
    # The label will be translated only when it's actually rendered
```

## 🗺️ URL-Based Locale Selection

In addition to the `Accept-Language` header, you can allow users to switch languages via a URL prefix or a query parameter.

### 1. URL Prefixes
Configure your router to capture the locale in the path.

```python
# app.py
from eden.middleware import LocaleMiddleware

app.add_middleware(LocaleMiddleware, use_path=True)

@app.get("/{locale}/dashboard")
async def dashboard(request, locale: str):
    # Locale is automatically set for this request
    return request.render("dashboard.html")
```

### 2. The Language Switcher (Template)
Creating a simple menu to allow users to change their language.

```html
<div class="language-switcher">
    <a href="?lang=en" class="{{ 'font-bold' if current_locale == 'en' }}">English</a>
    <a href="?lang=fr" class="{{ 'font-bold' if current_locale == 'fr' }}">Français</a>
</div>
```

## Best Practices

- **Placeholders**: Always use named placeholders (e.g., `{name}`) instead of positional ones for better translator context.
- **Lazy Loading**: Use `_l` for static definitions like model labels and error messages.
- **Fallbacks**: Ensure `DEFAULT_LOCALE` is always set to a complete translation set (usually English).
